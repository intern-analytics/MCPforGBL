[[[ SCHEMA PREFIX RULE ]]]
- Unicommerce and EBO tables live in the "DataWarehouse" schema and must always be referenced with that prefix.
- The Myntra SJIT crawler table lives in the public schema.
- Quick-commerce raw tables live in the public schema: public.zepto_sales, public.zepto_inventory, public.blinkit_sales, public.blinkit_soh, public.instamart_sale, public.instamart_inventory.
- Never reference a table without its schema prefix.

[[[ ANALYST LOGIC & DATE HANDLING ]]]
1. DEFAULT DATE FLOOR: Filter for dates >= '2024-01-01' unless requested otherwise. Use 'YYYY-MM-DD' format.
2. CASTING RULES:
   - Wrap timestamptz in DATE().
   - Wrap EBO BillDate in TO_DATE(val, 'DD/MM/YYYY').
   - Wrap Myntra created_on in DATE(val::timestamp).
   - Cast EBO Amount/Quantity ::numeric before sum.
   - Cast EBO Stock ::numeric before sum.
   - Cast Myntra final_amount ::numeric before sum.
3. EBO QUIRK: ALWAYS use LTRIM("EANCode", '''') for EBO Sales joins to strip leading apostrophes (Excel-export artifact from the upstream Wondersoft feed). chumbak_ebo_inventory."EanCode" does NOT have this issue — use it as-is.
4. QCOM REVENUE RULES (per channel):
   - Zepto: revenue = selling_price (numeric, direct). Units = sales_qty_units.
   - Blinkit: revenue = mrp * qty_sold (selling_price not present). Units = qty_sold.
   - Instamart: revenue = gmv (pre-computed). Units = units_sold. Date column = ordered_date.
5. LIMIT RESULTS: Always apply LIMIT 100 unless requested otherwise.
6. READ-ONLY: Reject any query containing INSERT, UPDATE, DELETE, DROP, ALTER, or TRUNCATE.
7. NEVER SELECT *: List only the columns needed for the task.
8. NEVER GUESS COLUMN NAMES: DataWarehouse columns are case-sensitive and must be wrapped in double quotes. Public schema qcom and Myntra-crawler column names are lowercase and do NOT need quoting.
9. PII: Do not query or return PII (names, addresses, phone numbers, email, GSTIN, customer mobile) unless explicitly required and authorised. Avoid: "Shipping Address *", "Billing Address *", "Notification Email", "Notification Mobile", "CustomerMobile", "CustomerName", "Email".
10. OFFLINE CONTEXT: If the user asks for 'offline' sales, 'offline' units, or 'offline' inventory, you MUST use the EBO tables ("DataWarehouse".chumbak_ebo_sales and "DataWarehouse".chumbak_ebo_inventory). Do NOT use Unicommerce for offline retail data.

[[[ MANDATORY SALEORDERS FILTERS ]]]
EVERY query against 'chumbak_in_unicommerce_saleorders' MUST include these filters:
- so."Sale Order Item Status" NOT IN ('CANCELLED')

  ELSE 'D2C'
END AS business_segment

[[[ CLEAN CHANNEL LOGIC ]]]
Use this CASE expression for grouping by channel:
CASE
  WHEN so."Channel Name" ILIKE 'AMAZON%' OR so."Channel Name" = 'amazon' OR so."Channel Name" ILIKE 'COCOBLU%' THEN 'Amazon'
  WHEN so."Channel Name" ILIKE 'FLIPKART%' THEN 'Flipkart'
  WHEN so."Channel Name" ILIKE 'MYNTRA%' THEN 'Myntra'
  WHEN so."Channel Name" ILIKE 'NYKAA%' THEN 'Nykaa'
  WHEN so."Channel Name" ILIKE 'SHOPIFY%' THEN 'Shopify'
  WHEN so."Channel Name" ILIKE 'NOON%' OR so."Channel Name" ILIKE 'NAMSHI%' THEN 'Noon'
  WHEN so."Channel Name" ILIKE 'AJIO%' THEN 'Ajio'
  WHEN so."Channel Name" ILIKE 'TATA_CLIQ%' OR so."Channel Name" ILIKE 'TATACLIQ%' THEN 'Tata Cliq'
  WHEN so."Channel Name" ILIKE 'JIOMART%' THEN 'Jiomart'
  WHEN so."Channel Name" ILIKE 'CRED%' THEN 'Cred'
  WHEN so."Channel Name" ILIKE 'PEPPERFRY%' THEN 'Pepperfry'
  ELSE 'Others'
END AS clean_channel

Use clean_channel and business_segment as the primary dimensions for reporting. Namshi rolls into Noon; Cocoblu rolls into Amazon. B2B and Mall segments are now included in the raw data but should be grouped accordingly.

[[[ PER-TABLE COLUMN REFERENCE ]]]

(1) "DataWarehouse".chumbak_in_unicommerce_saleorders — Online marketplace orders, revenue, channel analysis, fulfilment status, returns.
    - Join key: "Item SKU Code" (character varying) — primary SKU key for joining to inventory, PO, GRN, gatepass, itemmaster, qcom.
    - Cross-source EAN key: "Seller SKU Code" — holds the 13-digit EAN barcode for EBO-relevant SKUs; used to join to EBO sales/inventory.
    - Fallback EAN: "Item Type EAN" — used only for itemmaster fallback joins.
    - Alternate join key: "AlternateProductCodes" (character varying) — use this to join to itemmaster."EAN" if "Item SKU Code" mapping fails.
    - Date column: "Order Date as dd/mm/yyyy hh:MM:ss" (timestamp with time zone) — ALWAYS wrap in DATE().
    - Channel column: "Channel Name" — use with the clean_channel CASE expression for grouping; raw only for sub-channel granularity.
    - Status column: "Sale Order Item Status" — must exclude CANCELLED.
    - Order identifier: "Sale Order Code" — use COUNT(DISTINCT) for order counts.
    - Quantity column: Each row is a unit. Use COUNT(*) for online unit quantity.

(2) "DataWarehouse".chumbak_in_unicommerce_inventorysnapshot — Shelf-level warehouse inventory, blocked/damaged stock, facility tracking.
    - Join key: "Item Type SKU Code" (character varying).
    - Date column: "Inventory Snapshot Date" (date) — compare directly, no wrap. MANDATORY: inv."Inventory Snapshot Date" = (SELECT MAX("Inventory Snapshot Date") FROM "DataWarehouse".chumbak_in_unicommerce_inventorysnapshot).
    - MANDATORY Facility Filter: inv."Facility" NOT LIKE 'EBO-%' (EBO stock must be queried from chumbak_ebo_inventory).
    - Facility column: "Facility".
    - Stock math columns: "Quantity", "Quantity Blocked", "Quantity Damaged", "Quantity Not Found".
    - Available = "Quantity" - "Quantity Blocked" - "Quantity Damaged" - "Quantity Not Found".

(3) "DataWarehouse".chumbak_in_unicommerce_purchaseorders — Supply chain POs, received vs ordered, vendor performance.
    - Join key: "Item SkuCode" (character varying).
    - Date column: "Latest Updated" (timestamp with time zone) — wrap in DATE().
    - PO identifier: "PO Code" — use for joining to GRN.
    - Status column: "Purchase Order Status".
    - Vendor column: "Vendor Name".
    - Quantity columns: "Order Quantity", "Recieved Quantity" (TYPO INTENTIONAL — do not correct).
    - Ageing: "PO Ageing (Days)".
    - Fill rate = "Recieved Quantity"::float / NULLIF("Order Quantity", 0). Pending = "Order Quantity" - "Recieved Quantity".

(4) "DataWarehouse".chumbak_in_unicommerce_itemmaster — Product dimension: metadata, descriptions, dimensions, pricing, category/department.
    - Primary join: "Product Code" (character varying).

(5) "DataWarehouse".chumbak_in_unicommerce_grn — Goods Receipt Notes: received qty, rejections, vendor quality, inwarding.
    - Join key: "Item SkuCode" (character varying).
    - Date column: "GRN Date" (timestamp with time zone) — wrap in DATE().
    - Status column: "Grn item Status".
    - Vendor column: "Vendor Name".
    - Quantity columns: "Quantity Received", "Quantity Rejected", "Rejected Qty Inwarded" (typo intentional), "Tolerance Qty Inwarded" (typo intentional).
    - PO linkage: "PO Code" — join to purchaseorders."PO Code".
    - Rejection rate = "Quantity Rejected"::float / NULLIF("Quantity Received" + "Quantity Rejected", 0).

(6) "DataWarehouse".chumbak_in_unicommerce_gatepass — Outward movements, inter-facility transfers, returns to vendors, lost stock.
    - Join key: "Item SkuCode" (character varying).
    - Date column: "Gatepass Created At" (timestamp with time zone) — wrap in DATE().

[[[ JOIN PATTERNS ]]]
Prefer SKU-based joins. Use EAN only as a fallback when joining to itemmaster or when crossing the Unicommerce <-> EBO boundary.

-- Sales <-> Shelf Inventory
ON so."Item SKU Code" = inv."Item Type SKU Code"

-- Sales <-> Purchase Orders
ON so."Item SKU Code" = po."Item SkuCode"

-- Sales <-> GRN
ON so."Item SKU Code" = grn."Item SkuCode"

-- Sales <-> Gatepass
ON so."Item SKU Code" = gp."Item SkuCode"

-- Sales <-> Item Master (SKU, preferred)
ON so."Item SKU Code" = im."Product Code"

-- Sales <-> Item Master (Alternate Product Code to EAN fallback)
ON so."AlternateProductCodes" = im."EAN"

-- Sales <-> Item Master (EAN, fallback only)
ON so."Item Type EAN" = im."EAN"

-- Sales <-> Myntra SJIT crawler
ON so."Item SKU Code" = myn.seller_sku_code

-- Sales <-> EBO Sales (PREFERRED: EAN with mandatory apostrophe strip)
ON so."Seller SKU Code" = LTRIM(ebo."EANCode", '''')

-- Sales <-> EBO Inventory (EanCode is clean, no apostrophe)
ON so."Seller SKU Code" = ebo_inv."EanCode"

-- Sales <-> Zepto Sales (EAN-based)
ON so."Seller SKU Code" = zs.ean AND zs.brand_name IN ('Chumbak')

-- Sales <-> Zepto Inventory (EAN-based)
ON so."Seller SKU Code" = zi.ean AND zi.brand_name = 'Chumbak'

-- Zepto Sales <-> Zepto Inventory
ON zs.sku_number = zi.sku_code AND zs.brand_name = zi.brand_name

-- Sales <-> Blinkit Sales (item_name pattern match — no EAN available)
ON so."Item SKU Code" = bs.item_id AND bs.brand IN ('chumbak')  -- prefer item_id if mapped; otherwise join via item_name

-- Blinkit Sales <-> Blinkit SOH
ON bs.item_id = bsoh.item_id AND bs.brand = bsoh.brand

-- Sales <-> Instamart Sales (item_code)
ON so."Item SKU Code" = ins.item_code AND ins.brand IN ('chumbak')

-- Instamart Sales <-> Instamart Inventory
ON ins.item_code = ii.sku_code AND ins.brand = ii.brand

-- PO <-> GRN (reconciliation)
ON po."PO Code" = grn."PO Code" AND po."Item SkuCode" = grn."Item SkuCode"

-- EBO Sales <-> EBO Inventory
ON ebo_s."ProductCode" = ebo_i."ProductCode" AND ebo_s."StoreCode" = ebo_i."StoreCode"

[[[ COMMON METRIC PATTERNS ]]]
- Distinct online order count: COUNT(DISTINCT so."Sale Order Code")
- Online unit quantity (Unicommerce): COUNT(*)
- Available shelf inventory: inv."Quantity" - inv."Quantity Blocked" - inv."Quantity Damaged" - inv."Quantity Not Found"
- PO fill rate: po."Recieved Quantity"::float / NULLIF(po."Order Quantity", 0)
- PO pending quantity: po."Order Quantity" - po."Recieved Quantity"
- GRN rejection rate: grn."Quantity Rejected"::float / NULLIF("Quantity Received" + grn."Quantity Rejected", 0)
- EBO offline revenue: SUM(ebo."Amount"::numeric)
- EBO offline units: SUM(ebo."Quantity"::numeric)
- EBO store-level stock: SUM(ebo_inv."Stock"::numeric)
- Myntra SJIT final amount: SUM(myn.final_amount::numeric)
- Zepto revenue: SUM(zs.selling_price)
- Zepto units: SUM(zs.sales_qty_units)
- Zepto platform inventory: SUM(zi.units)
- Blinkit revenue: SUM(bs.mrp * bs.qty_sold)
- Blinkit units: SUM(bs.qty_sold)
- Blinkit SOH (total): SUM(bsoh.backend_inv_qty + bsoh.frontend_inv_qty)
- Instamart revenue: SUM(ins.gmv)
- Instamart units: SUM(ins.units_sold)
- Instamart platform inventory: SUM(ii.warehouse_qty_available)
- Combined Qcom revenue (CASE): SUM(CASE WHEN channel = 'Zepto' THEN zs.selling_price WHEN channel = 'Blinkit' THEN bs.mrp * bs.qty_sold WHEN channel = 'Instamart' THEN ins.gmv END) — use UNION ALL approach when combining across qcom channels.

NOTE: "Recieved Quantity" is the actual column name (intentional typo — do not correct). Same for "Rejected Qty Inwarded" and "Tolerance Qty Inwarded".
